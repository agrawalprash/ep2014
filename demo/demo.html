UI layer

What is it exactly?

  • A bridge to allow writing live HTML apps for Python
  • Declarative way of writing UIs
  • Automatic binding between the model and the view
  • UI development can be easily handed off to an HTML/CSS/JS guru

What it is NOT

  • Yet another UI toolkit
  • An HTML widget library
  • Django/Flask

How we're doing it

  • Python model on the left
  • Qt's QWebViewer widget - browser within Qt
  • Javascript proxies for Python models
  • AngularJS for Javascript side MVC

Things to note

  • Proxies for primitive variables, lists, instances
  • Method call
